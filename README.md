# drstrange-process

Textbook ingestion pipeline for **MediFact**, a medical note validation system. Converts medical textbook PDFs into structured, chunked vector data in [Zilliz Cloud](https://cloud.zilliz.com) (Milvus) for hybrid semantic + lexical retrieval.

## Current Data

Unified `textbook_chunks` collection in Zilliz Cloud with **~21,900 chunks** across two textbooks:

| Book | Pages | Chapters | Chunks | Embedding Time |
|------|-------|----------|--------|----------------|
| Harrison's Principles of Internal Medicine, 21st Ed. | 3,825 | 491 | 20,799 | ~42 min (CPU) |
| Bates' Pocket Guide to Physical Examination | 481 | 20 | 1,142 | ~2 min (CPU) |

All chunks share a single collection, distinguished by the `book` field (`"harrisons21st"` or `"bates"`). Supports cross-book semantic search with scores >0.92 on medical queries.

## Pipeline

```
<book>.pdf
  --> pipeline/extract_toc.py               TOC + metadata extraction
  --> pipeline/run_docstrange_textbook.py    GPU-accelerated OCR (DocStrange/Nanonets-OCR2-3B)
  --> pipeline/postprocess_docstrange.py     7-pass cleanup, classification, structuring
  --> pipeline/chunk_textbook.py             Content-type-aware chunking for retrieval
  --> pipeline/ingest_milvus.py              Embedding + Zilliz Cloud insertion
```

For large PDFs (>1000 pages), use `pipeline/run_chunked_ocr.py` instead of `run_docstrange_textbook.py` to split the book into page-range chunks and process them in parallel.

| Stage | Input | Output | Time (Harrison's) | Time (Bates) |
|-------|-------|--------|--------------------|--------------|
| TOC extraction | PDF | `.toc.json`, `.meta.json` | <1s | <1s |
| OCR conversion | PDF | Markdown | ~81h (A100, parallel=2) | ~3.3h (RTX 4000) |
| Postprocessing | Markdown | PageBlocks (JSON) | <30s | <10s |
| Chunking | PageBlocks | Chunks (JSON) | <10s | <5s |
| Ingestion | Chunks | Zilliz Cloud collection | ~4 min (CPU) | ~20s (CPU) |

## Milvus Collection Schema

```
textbook_chunks
  id              VARCHAR PK         "harrisons21st_ch275_p2145_c0003"
  text            VARCHAR (BM25)     chunk text, analyzer-enabled
  dense_vector    FLOAT_VECTOR 768   S-PubMedBert-MS-MARCO (IP-normalized)
  sparse_vector   SPARSE_FLOAT_VECTOR  auto-generated by BM25 Function
  book            VARCHAR            "harrisons21st" | "bates"
  chapter_num     INT16              chapter number
  chapter_title   VARCHAR
  section         VARCHAR
  subsection      VARCHAR
  page_start      INT16              book page number
  page_end        INT16
  chunk_index     INT16
  content_type    VARCHAR            prose|table|list|image_desc|clinical_criteria|...
```

## Setup

### RunPod (GPU pod for OCR -- RTX 4000 Ada or A100)

```bash
git clone https://github.com/keentechcodes/drstrange-process.git
cd drstrange-process
chmod +x scripts/setup_runpod.sh
./scripts/setup_runpod.sh    # installs venv, deps, pre-downloads OCR model (~6 GB)
source .venv/bin/activate
```

### Zilliz Cloud credentials

```bash
cp .env.example .env
# Edit .env with your Zilliz Cloud endpoint and token
```

## Usage

### Processing a new textbook

```bash
source .venv/bin/activate

# 1. Extract TOC
python pipeline/extract_toc.py HARRISONS21ST.pdf

# 2. Run OCR (use run_chunked_ocr.py for large books)
python pipeline/run_chunked_ocr.py HARRISONS21ST.pdf results_harrisons4 \
    --pages-per-chunk 500 --parallel 2

# 3. Postprocess
python pipeline/postprocess_docstrange.py \
    results_harrisons4/HARRISONS21ST.md \
    harrisons21st.toc.json harrisons21st.meta.json

# 4. Chunk
python pipeline/chunk_textbook.py \
    results_harrisons4/harrisons21st.pageblocks.json harrisons21st.meta.json \
    --output results_harrisons4/harrisons21st.chunks.json

# 5. Embed and ingest (appends to existing collection)
python pipeline/ingest_milvus.py \
    results_harrisons4/harrisons21st.chunks.json \
    --embed-batch-size 128 --embed-cache results_harrisons4/embeddings.pkl
```

Use `--recreate` on the ingest step to drop and recreate the collection. Use `--embed-cache` to avoid recomputing embeddings on retries (~42 min saved).

## Key Design Decisions

- **Embedding model:** [S-PubMedBert-MS-MARCO](https://huggingface.co/pritamdeka/S-PubMedBert-MS-MARCO) -- PubMedBERT fine-tuned on MS-MARCO for medical information retrieval (768-dim, mean pooling, max 350 tokens)
- **Hybrid retrieval:** Dense vectors (IP) + BM25 sparse vectors with RRF reranking
- **Content-type-aware chunking:** Different strategies for prose, tables, exam techniques, lists, clinical criteria -- preserves document structure
- **Anti-hallucination OCR:** Strict prompt engineering + per-page validation prevents vision model from fabricating content
- **BM25 server-side:** Sparse vectors auto-generated by Milvus Function on insert, no client-side sparse encoding needed
- **Unified collection:** All textbooks share one collection, filtered by `book` field for book-specific queries

## Part of MediFact

This repo handles textbook ingestion, one component of the MediFact system. The retrieval and NLI validation layers (consuming this vector data to check medical note accuracy) are the next phase.
