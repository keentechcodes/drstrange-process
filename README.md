# drstrange-process

Textbook ingestion pipeline for **VeriMed**, a medical note validation system. Converts medical textbook PDFs into structured, chunked vector data in [Zilliz Cloud](https://cloud.zilliz.com) (Milvus) for hybrid semantic + lexical retrieval.

Currently ingests **Bates' Pocket Guide to Physical Examination** (481 PDF pages, 20 chapters) into 1,142 searchable chunks with 768-dim dense embeddings and BM25 sparse vectors.

## Pipeline

```
BATES.pdf
  --> extract_toc.py           TOC + metadata extraction
  --> run_docstrange_textbook.py   GPU-accelerated OCR (DocStrange/Nanonets-OCR2-3B)
  --> postprocess_docstrange.py    7-pass cleanup, classification, structuring
  --> chunk_textbook.py            Content-type-aware chunking for retrieval
  --> ingest_milvus.py             Embedding + Zilliz Cloud insertion
```

| Stage | Input | Output | Time |
|-------|-------|--------|------|
| TOC extraction | PDF | `bates.toc.json`, `bates.meta.json` | <1s |
| OCR conversion | PDF (481 pages) | Markdown (17K lines, 807K chars) | ~3.3h (GPU) |
| Postprocessing | Markdown | 3,961 PageBlocks (JSON) | <10s |
| Chunking | PageBlocks | 1,142 chunks (JSON) | <5s |
| Ingestion | Chunks | Zilliz Cloud collection | ~19s (GPU) |

## Milvus Collection Schema

```
textbook_chunks
  id              VARCHAR PK         "bates_ch16_p0293_c0003"
  text            VARCHAR (BM25)     chunk text, analyzer-enabled
  dense_vector    FLOAT_VECTOR 768   S-PubMedBert-MS-MARCO (IP-normalized)
  sparse_vector   SPARSE_FLOAT_VECTOR  auto-generated by BM25 Function
  book            VARCHAR            partition key ("bates")
  chapter_num     INT16              1-20
  chapter_title   VARCHAR
  section         VARCHAR
  subsection      VARCHAR
  page_start      INT16              book page number
  page_end        INT16
  chunk_index     INT16
  content_type    VARCHAR            prose|table|list|exam_technique|image_desc|clinical_criteria
```

## Setup

### RunPod (GPU pod — RTX 4000 Ada or similar)

```bash
git clone https://github.com/keentechcodes/drstrange-process.git
cd drstrange-process
chmod +x setup_runpod.sh
./setup_runpod.sh        # installs venv, deps, pre-downloads OCR model (~6 GB)
source .venv/bin/activate
```

### Zilliz Cloud credentials

```bash
cp .env.example .env
# Edit .env with your Zilliz Cloud endpoint and token
```

## Usage

```bash
source .venv/bin/activate

# 1. Extract TOC (needs BATES.pdf on the pod)
python extract_toc.py BATES.pdf

# 2. Run OCR (~3.3h on RTX 4000 Ada)
python run_docstrange_textbook.py BATES.pdf results_book1

# 3. Postprocess
python postprocess_docstrange.py results_book1/BATES.md bates.toc.json bates.meta.json

# 4. Chunk
python chunk_textbook.py results_book1/BATES.pageblocks.json bates.meta.json \
    --output results_book1/BATES.chunks.json

# 5. Embed and ingest into Zilliz Cloud
python ingest_milvus.py results_book1/BATES.chunks.json
```

## Key Design Decisions

- **Embedding model:** [S-PubMedBert-MS-MARCO](https://huggingface.co/pritamdeka/S-PubMedBert-MS-MARCO) , PubMedBERT fine-tuned on MS-MARCO for medical information retrieval (768-dim, mean pooling, max 350 tokens)
- **Hybrid retrieval:** Dense vectors (IP) + BM25 sparse vectors with RRF reranking
- **Content-type-aware chunking:** Different strategies for prose, tables, exam techniques, lists, clinical criteria which preserves document structure
- **Anti-hallucination OCR:** Strict prompt engineering + per-page validation prevents vision model from fabricating content
- **BM25 server-side:** Sparse vectors auto-generated by Milvus Function on insert where no client-side sparse encoding is needed
